C# Coding Style

This is not an all inclusive document on coding style that should be used. The purpose of this document is to cover the most common situations and show the general patterns. For frequent situations that are not contained in this document they will be decided then.

  1. All indention will be done with tabs. Spaces are not ment to be used for indention.

  2. All class, function, and variable names will be descriptive. They should accurately describe what it refers to. This means that variable/object/class names should be noun (subject) based and functions should be verb (predicate) based. Variables should not contain abbreviations, this is to prevent confusion between meanings of the abbreviation. Temporary variables however can be abbreviated. Thus tmp is ok to use but delItem should not be used instead of deleteItem

  3. All local variable names will be camelCase.

    
void createConversation();
int createId;

  4. All member variables will begin with an "_" and use camelCase.

    
private int _count;

  5. All properties will refer to one or more member variables.

  6. All class, property, and function names will be PascalCase.

  7. Opening braces should never be on a new line, instead always place them at the end of the previous line with a space between the brace and the previous code.

  8. Closing braces should always be on a line of their own. If, for example, you have an if else statement it should be like this: 

    
if (x == y) {
	return x * y;
}
else if (y != 0) {
	return x / y;
}
    The reason that this is the case is to improve readability when using folding. This way you can fold individual elements of the if statement (or other statement) and not get confused as to what code relates to what condition.

  9. All binary operators (+, -, *, /, &&, ||, ...) must have a space on either side of the operator.

  10. All unary operators must be touching the value that they operate on. Ex: i++

  11. There will never be a space on the inside of parentheses. Ex: (x + 3) / 2;
