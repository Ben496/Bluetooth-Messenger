Requirements
Bluetooth Messenger

Desktop UI Requirements

  1. Name: bluetooth_contact_syncing
    Type: functional
    Summary: Be able to sync with app via bluetooth and send/receive any contact adds/edits/deletions through desktop held contact list

  2. Name: bluetooth_message_syncing
    Type: functional
    Summary: Be able to sync with app via bluetooth and receive all messages from two weeks or later

  3. Name: display_contacts
    Type: functional
    Summary: Be able to display all contacts received from phone

  4. Name: search_contacts
    Type: functional
    Summary: Be able to search by name all contacts received from phone

  5. Name: bluetooth_message_syncing
    Type: functional
    Summary: Be able to sync with app via bluetooth and receive all messages from two weeks or before

  6. Name: add_contact
    Type: functional
    Summary: Add contact to desktop held contact listing file

  7. Name: delete_contact
    Type: functional
    Summary: Delete contact from desktop held contact listing file

  8. Name: manage_contact_fields
    Type: functional
    Summary: be able to manage all potential fields of a contact

  9. Name: message_receive
    Type: functional
    Summary: Be able to receive individual messages from app within 1 second of phone receiving message

  10. Name: message_send
    Type: functional
    Summary: Be able to send individual messages (sms or mms) to app within 1 second of desktop UI send key press

  11. Name: message_send_error
    Type: functional
    Summary: Notify user if there was an error sending message.

  12. Name: message_resend
    Type: functional
    Summary: Be able to resend a message

  13. Name: message_forward
    Type: functional
    Summary: Be able to forward a message to a person

  14. Name: message_notification
    Type: functional
    Summary: display notification of received messages

  15. Name: minimize_to_taskbar
    Type: functional
    Summary: Be able to minimize to taskbar when running

  16. Name: emoji_handling
    Type: functional
    Summary: Enable handling for emojis in text messages

  17. Name: multimedia_handling
    Type: functional
    Summary: Enable handling of multimedia messages (mms) being received and sent

  18. Name: Windows10_combatitibility
    Type: constraint
    Summary: Must be combatible with Windows 10

  19. Name: xml_message_compliance
    Type: constraint
    Summary: Must comply with standard messaging protocol (i.e. XML) so that app can send text messages over network correctly

  20. Name: connection_password
    Type: functional
    Summary: User must enter a password to initiate a connection to the phone each time the phone is connected

  21. Name: connection_password_characters
    Type: constraint
    Summary: Must support alphanumeric and special characters up to 100 characters in length

  22. Name: connection_password_validation
    Type: risk
    Summary: Must send password information in a secure way

Mobile App Requirements

  1. Name: bluetooth_contact_syncing
    Type: functional
    Summary: Be able to sync phone's contacts with desktop gui via bluetooth and update phone's contact book with any contact adds/edits/deletions

  2. Name: bluetooth_message_syncing
    Type: functional
    Summary: Be able to sync with desktop gui via bluetooth and send all messages from two weeks or later

  3. Name: receive_messages
    Type: functional
    Summary: Be able to forward received messages to the desktop gui within 1 second of the phone getting the message

  4. Name: send_messages
    Type: functional
    Summary: Be able to send from the phone messages obtained from the desktop gui

  5. Name: connection_password
    Type: functional
    Summary: Must be able to set and change the connection password

  6. Name: connection_password_storage
    Type: risk
    Summary: Must not store the password and instead store the sha512 hash of the password or store the password in a secure way

  7. Name: android_5.0_compatability
    Type: constraint
    Summary: Must run on Android 5.0 Lollipop

  8. Name: power_conservation
    Type: constraint
    Summary: Must be able to conserve power between messages
