Desktop UI Requirements

    Display

  1. Name: message_entry_field
    Type: functional
    Summary: Field to enter text for outgoing sms/mms message.

  2. Name: initiate_message_send (Updated)
    Type: functional
    Summary: Initiate sending message.

  3. Name: initiate_message_resend (Updated)
    Type: functional
    Summary: Initiate resending message in case of failure

  4. Name: initiate_message_forward (Updated)
    Type: functional
    Summary: Initiate forwarding message to a different contact

  5. Name: initiate_attach_multimedia (Updated)
    Type: functional
    Summary: Initiate attaching multimedia to message (i.e. sound, pic, video)

  6. Name: initiate_manual_sync (Updated)
    Type: functional
    Summary: Initiate a manually sync between desktop and mobile app

  7. Name: display_conversation
    Type: functional
    Summary: Displays messages from history (up to 100) between selected contact from history

  8. Name: contact_selection_menu
    Type: functional
    Summary: Menu to select contact to message

  9. Name:display_current_contact_name
    Type: functional
    Summary: Displays current contact in conversation

  10. Name: message_notification
    Type: functional
    Summary: display notification of received messages

  11. Name: session_status
    Type: functional
    Summary: Show session status

  12. Name: initiate_session_termination (Updated)
    Type: functional
    Summary: Terminates app/desktop connectivity unless password supplied again

  13. Name: initiate_media_request (New)
    Type: functional
    Summary: Requests media from the phone.

    14. Messaging

  14. Name: message_error_handling
    Type: constraint
    Summary: Detect if message did not send correctly.

  15. Name: message_send_error
    Type: functional
    Summary: Notify user if there was an error sending message.

  16. Name: bluetooth_message_syncing
    Type: constraint
    Summary: Be able to sync with app via bluetooth and receive all messages from two weeks or before

  17. Name: message_receive
    Type: constraint
    Summary: Be able to receive individual messages from app within 1 second of phone receiving message

  18. Name: message_send
    Type: constraint
    Summary: Be able to send individual messages (sms or mms) to app within 1 second of desktop UI send key press

  19. Name: message_resend_handling
    Type: constraint
    Summary: Be able to resend a message

  20. Name: message_forward_handling
    Type: constraint
    Summary: Be able to forward a message to a person

  21. Name: multimedia_handling
    Type: constraint
    Summary: Enable handling of multimedia messages (mms) being received and sent

  22. Name: recent_messages_storage
    Type: constraint
    Summary: Storage structure for up to 100 text messages within individual conversations

  23. Name: conversation_storage
    Type: constraint
    Summary: Storage structure for up to 100 conversations.

    24. Contact Management

  24. Name: display_contacts
    Type: functional
    Summary: Be able to display all contacts received from phone

  25. Name: search_contacts
    Type: functional
    Summary: Be able to search by name all contacts received from phone

  26. Name: add_contact
    Type: functional
    Summary: Add contact to desktop held contact listing file

  27. Name: delete_contact
    Type: functional
    Summary: Delete contact from desktop held contact listing file

  28. Name: manage_contact_fields
    Type: functional
    Summary: be able to manage all potential fields of a contact

  29. Name: bluetooth_contact_syncing
    Type: constraint
    Summary: Be able to sync with app via bluetooth and send/receive any contact adds/edits/deletions through desktop held contact list

    30. Misc.

  30. Name: minimize_to_taskbar
    Type: functional
    Summary: Be able to minimize to taskbar when running

  31. Name: connection_password
    Type: functional
    Summary: User must enter a password to initiate a connection to the phone each time the phone is connected

  32. Name: Windows10_combatitibility
    Type: constraint
    Summary: Must be combatible with Windows 10

  33. Name: xml_message_compliance
    Type: constraint
    Summary: Must comply with standard messaging protocol (i.e. XML) so that app can send text messages over network correctly

  34. Name: connection_password_characters
    Type: constraint
    Summary: Must support alphanumeric and special characters up to 100 characters in length

  35. Name: hotkey_support
    Type: constraint
    Summary: Allow for support of hotkeys for common functions.

  36. Name: auto_disconnect
    Type: constraint
    Summary: Upon bluetooth disconnection, password will be needed again

  37. Name: bluetooth_ping
    Type: constraint
    Summary: Pings desktop UI to check if connection still active

  38. Name: connection_password_validation
    Type: risk
    Summary: Must send password information in a secure way
