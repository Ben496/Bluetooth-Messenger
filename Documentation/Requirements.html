Requirements
Bluetooth Messenger

Desktop UI Requirements

  1. Name: bluetooth_contact_syncing
    Type: functional
    Summary: Be able to sync with app via bluetooth and send/receive any contact adds/edits/deletions through desktop held contact list

  2. Name: bluetooth_message_syncing
    Type: functional
    Summary: Be able to sync with app via bluetooth and receive all messages from two weeks or before

  3. Name: display_contacts
    Type: functional
    Summary: Be able to display all contacts received from phone

  4. Name: bluetooth_message_syncing
    Type: functional
    Summary: Be able to sync with app via bluetooth and receive all messages from two weeks or before

  5. Name: add_contact
    Type: functional
    Summary: Add contact to desktop held contact listing file

  6. Name: delete_contact
    Type: functional
    Summary: Delete contact from desktop held contact listing file

  7. Name: manage_contact_fields
    Type: functional
    Summary: be able to manage all potential fields of a contact

  8. Name: message_receive
    Type: functional
    Summary: Be able to receive individual messages from app within 1 second of phone receiving message

  9. Name: message_send
    Type: functional
    Summary: Be able to send individual messages to app within 1 second of desktop UI send key press

  10. Name: message_notification
    Type: functional
    Summary: display notification of received messages

  11. Name: minimize_to_taskbar
    Type: functional
    Summary: Be able to minimize to taskbar when running

  12. Name: emoji_handling
    Type: functional
    Summary: Enable handling for emojis in text messages

  13. Name: multimedia_handling
    Type: functional
    Summary: Enable handling of multimedia messages being received and sent

  14. Name: Windows10_combatitibility
    Type: constraint
    Summary: Must be combatible with Windows 10

  15. Name: xml_message_compliance
    Type: constraint
    Summary: Must comply with standard messaging protocol (i.e. XML) so that app can send text messages over network correctly

  16. Name: secure_bluetooth_messages
    Type: risk
    Summary: Must have some form of password between app and desktop gui so that bluetooth pairing is not only obstable to seeing messages.
